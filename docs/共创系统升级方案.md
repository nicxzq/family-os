## 三、E0-B · 微信开放平台扫码登录

### 3.1 与 E0-A 的区别

| 维度 | E0-A（公众号 OAuth，已暂停）| E0-B（开放平台扫码，本次实现）|
|---|---|---|
| 平台 | mp.weixin.qq.com | open.weixin.qq.com |
| 适用浏览器 | 仅微信内置浏览器 | 任意浏览器（PC / Safari / Chrome / 微信）|
| 用户操作 | 静默跳转（无感知）| 看到二维码 → 用手机微信扫码 |
| AppID | 公众号 AppID | 开放平台网站应用 AppID |
| 获取的标识 | openid（公众号维度）| unionid（跨平台统一 ID）|

### 3.2 完整登录流程

```
用户在任意浏览器点「微信扫码登录」
        │
        ▼
 重定向到微信开放平台扫码页
 https://open.weixin.qq.com/connect/qrconnect
   ?appid=OPEN_APPID
   &redirect_uri=https://family.carlxu.cn/api/wechat-open-callback
   &response_type=code
   &scope=snsapi_login
   &state=RANDOM_STATE
        │
        ▼ 微信展示二维码，用户用手机扫码并确认
        │
 /api/wechat-open-callback?code=XXX&state=YYY
 (Vercel Serverless Function)
        │
        ├─ 用 code 换 access_token + unionid
        │
        ├─ 查 wechat_bindings（按 unionid）
        │      │
        │      ├─ 找到 → 生成 Supabase magic link → 跳回首页（自动登录）
        │      │
        │      └─ 没找到（新用户）→ 创建账号 + 绑定 unionid → 跳 /login?setup=1
        │
        ▼
 用户在 setup overlay 选择家庭角色 → 完成
```

### 3.3 你需要配置的地方

#### Step 1 · 微信开放平台

登录 [open.weixin.qq.com](https://open.weixin.qq.com)

1. **注册/登录**：用你的微信扫码登录开放平台
2. **创建网站应用**：管理中心 → 网站应用 → 创建网站应用
   - 应用名称：家族 OS / Family OS
   - 应用网址：`https://family.carlxu.cn`
   - 授权回调域：`family.carlxu.cn`（不带 https://，不带路径）
3. **等待审核**（通常 1-3 个工作日）
4. 审核通过后，记下：
   - AppID（开放平台网站应用 AppID，以 `wx` 开头，与公众号 AppID 不同）
   - AppSecret

#### Step 2 · Vercel 环境变量

新增两条（与现有公众号变量并存，名称不同）：

| 变量名 | 值 | 说明 |
|---|---|---|
| `WECHAT_OPEN_APPID` | 开放平台网站应用 AppID | 与 `WECHAT_APPID`（公众号）不同 |
| `WECHAT_OPEN_APPSECRET` | 开放平台 AppSecret | 保密，不进代码仓库 |

`SUPABASE_SERVICE_ROLE_KEY` 已有，无需重复添加。

### 3.4 技术实现（Codex 负责）

#### 文件改动清单

| 文件 | 变更 | 说明 |
|---|---|---|
| `supabase/schema.sql` | 追加列 | `wechat_bindings` 表增加 `unionid text` 列，作为主键（替代 openid）|
| `api/wechat-open-callback.js` | 新增 | 开放平台回调：code → unionid → Supabase session |
| `login.html` | 改动 | 微信按钮改为指向 `/api/wechat-open-login`；取消隐藏逻辑（任意浏览器显示）|

#### `supabase/schema.sql` 修改

将 `wechat_bindings` 主键从 `openid` 改为 `unionid`，因为开放平台返回 `unionid`（跨公众号/小程序唯一）：

```sql
-- 删除旧表，重建（或 alter + migrate）
-- 因为目前没有生产数据，直接 drop + recreate 最干净

drop table if exists public.wechat_bindings cascade;

create table if not exists public.wechat_bindings (
  unionid     text primary key,          -- 开放平台 unionid（跨应用唯一）
  openid      text,                      -- 可选：记录当次 openid，供调试
  user_id     uuid not null references public.profiles(id) on delete cascade,
  created_at  timestamptz default now()
);

alter table public.wechat_bindings enable row level security;
-- 无面向客户端的 policy，service_role 直接操作
```

#### `api/wechat-open-callback.js` 核心逻辑

```js
// api/wechat-open-callback.js
const SUPABASE_URL = 'https://zdmhnwxwblpzcjgszqpj.supabase.co';
const SITE = 'https://family.carlxu.cn';

async function sb(path, opts = {}) {
  const key = process.env.SUPABASE_SERVICE_ROLE_KEY;
  const res = await fetch(`${SUPABASE_URL}${path}`, {
    ...opts,
    headers: {
      apikey: key, Authorization: `Bearer ${key}`,
      'Content-Type': 'application/json', ...(opts.headers || {}),
    },
  });
  const text = await res.text();
  return text ? JSON.parse(text) : null;
}

module.exports = async function handler(req, res) {
  const { code } = req.query;
  if (!code) return res.redirect(`${SITE}/?error=wechat_no_code`);

  try {
    // 1. code → access_token + unionid
    const token = await fetch(
      `https://api.weixin.qq.com/sns/oauth2/access_token` +
      `?appid=${process.env.WECHAT_OPEN_APPID}` +
      `&secret=${process.env.WECHAT_OPEN_APPSECRET}` +
      `&code=${code}&grant_type=authorization_code`
    ).then(r => r.json());

    if (!token.unionid) return res.redirect(`${SITE}/?error=wechat_auth_failed`);
    const { unionid, openid } = token;

    // 2. 查 wechat_bindings
    const bindings = await sb(
      `/rest/v1/wechat_bindings?unionid=eq.${encodeURIComponent(unionid)}&select=user_id`
    );

    if (Array.isArray(bindings) && bindings.length > 0) {
      // 老用户 → 生成 magic link 登录
      const user = await sb(`/auth/v1/admin/users/${bindings[0].user_id}`);
      const link = await sb('/auth/v1/admin/generate_link', {
        method: 'POST',
        body: JSON.stringify({ type: 'magiclink', email: user.email, redirect_to: `${SITE}/` }),
      });
      return res.redirect(link.action_link);
    }

    // 新用户 → 创建账号 + 绑定 + 跳 setup
    const email = `wx_${unionid.slice(0, 18)}@fo.internal`;
    const newUser = await sb('/auth/v1/admin/users', {
      method: 'POST',
      body: JSON.stringify({
        email, email_confirm: true,
        user_metadata: { name: '新成员', avatar: 'chick', color: 'coral', role: 'child', family_role: '' },
      }),
    });
    if (!newUser?.id) return res.redirect(`${SITE}/?error=create_user_failed`);

    await sb('/rest/v1/wechat_bindings', {
      method: 'POST',
      headers: { Prefer: 'return=minimal' },
      body: JSON.stringify({ unionid, openid, user_id: newUser.id }),
    });

    const link = await sb('/auth/v1/admin/generate_link', {
      method: 'POST',
      body: JSON.stringify({ type: 'magiclink', email, redirect_to: `${SITE}/login.html?setup=1` }),
    });
    return res.redirect(link.action_link);

  } catch (err) {
    console.error('wechat-open-callback error:', err);
    return res.redirect(`${SITE}/?error=server_error`);
  }
};
```

#### `login.html` 改动

1. 微信按钮 `href` 改为指向新的 `/api/wechat-open-login`（或直接构造 open.weixin.qq.com URL）
2. 去掉 `hidden` 属性和 UA 检测——任意浏览器都可以显示此按钮
3. 按钮文字改为「微信扫码登录」

新增入口文件 `api/wechat-open-login.js`（重定向到开放平台扫码页）：

```js
module.exports = function (req, res) {
  const appid = process.env.WECHAT_OPEN_APPID;
  if (!appid) return res.status(500).send('WECHAT_OPEN_APPID not configured');
  const cb = encodeURIComponent('https://family.carlxu.cn/api/wechat-open-callback');
  const state = Math.random().toString(36).slice(2);
  res.redirect(302,
    `https://open.weixin.qq.com/connect/qrconnect?appid=${appid}&redirect_uri=${cb}&response_type=code&scope=snsapi_login&state=${state}#wechat_redirect`
  );
};
```

### 3.5 安全边界

- `WECHAT_OPEN_APPSECRET` 和 `SUPABASE_SERVICE_ROLE_KEY` **只能在 Vercel 环境变量里**，不进代码仓库
- `wechat_bindings` 表无面向客户端的 RLS policy，只有 service_role 可操作
- `state` 参数防 CSRF（此处用随机串，生产可加 cookie 校验）
