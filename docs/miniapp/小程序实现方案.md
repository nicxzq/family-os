# 好的家庭教育 · 微信小程序实现方案（交给 Codex）

> 目标：把现有静态站点 `family.carlxu.cn` 的内容，做成一个**微信小程序**。
> 约束：**个人主体**小程序（不能用 web-view）、**阿里云服务器**（可备案，做内容 API）。
> 配套：`miniapp/小程序原型.html` 是视觉与交互参照，**像素级对齐它**。

---

## 0. 给 Codex 的总指令（先读这段）

请用 **Taro 4 + React + TypeScript** 实现一个微信小程序，目录、页面、组件、数据结构、API 契约全部按本文件。
**视觉**严格参照 `miniapp/小程序原型.html` 与 `miniapp/data.js`（内容原文）、下方第 3 节的设计 token。
**不要**自己发明配色、字号、文案。**不要**引入 web-view。所有列表/卡片用原生组件实现，不要内联 SVG 做正文插画（用图片）。
完成后产出可在微信开发者工具直接运行的工程 + 一份 `README` 说明本地运行与发布步骤。

## 1. 技术选型与理由

| 选择 | 理由 |
|---|---|
| **Taro (React 语法)** | 现有站点是 React 心智，迁移成本最低；编译产物是标准小程序。 |
| **个人主体 → 纯原生页面** | 个人主体**不能用 `web-view`**，必须把每个网页改成小程序页面。 |
| **内容走阿里云 API** | 改文案/加内容时**不用重新提审**——只改数据。代码改动才需提审发布。 |
| **霞鹜文楷 `wx.loadFontFace`** | 整字库 >10MB，塞不进 2MB 主包；从阿里云 OSS 按需远程加载（失败回退系统字体）。 |
| **canvas 生成海报 + 保存相册** | 小程序**不能打印**；家庭墙改为生成图片存相册，再去冲印/打印。 |
| **原生「转发到聊天」** | 替代原站"把网址发给那个人"；每页配 `onShareAppMessage` 带 path 参数。 |

## 2. 信息架构（页面 = 原型里的 4 个 tab + 详情）

```
app
├─ tabBar
│   ├─ pages/ideas/index        六条     —— hero + 6 张想法卡
│   ├─ pages/persons/index      给家人   —— 4 张人物入口卡
│   ├─ pages/books/index        共读     —— 按人分组的书单
│   └─ pages/wall/index         家庭墙   —— 海报预览 + 保存相册/转发
└─ 非 tab（navigateTo 进入）
    ├─ pages/idea/index?no=1     想法详情（六条复用一页，按参数取数据）
    └─ pages/person/index?id=wife 人物分享页（4 人复用一页）
```

- 导航：tab 间用 `wx.switchTab`；进详情用 `wx.navigateTo`，详情页自带返回，**tabBar 自动隐藏**（与原型一致）。
- `idea` / `person` 详情页要实现 `onShareAppMessage`，转发卡片 `path` 指回该详情（带 `no` / `id`），对方点开**直接落在那一页**——这就是"单独发给那个人"。

```scss
// 字体（远程加载，回退系统衬线）

```
> 小程序单位用 **rpx**（750rpx = 屏宽）。原型里的 px 大致 ×2 转 rpx（如卡片标题 17px → 34rpx），按原型视觉微调即可。

### 字体加载（app 启动时）
```ts
// app.ts onLaunch
wx.loadFontFace({
  family: 'LXGW WenKai',
  source: 'url("https://你的备案域名/fonts/LXGWWenKai-Regular.woff2")',
  global: true,
  scope: 'webview', // Taro 下用全局，必要时各页再 loadFontFace 一次
  fail: () => {/* 静默回退到 Songti SC */}
});
```
> 字体文件放阿里云 OSS + CDN。建议**子集化**（只保留站内出现的汉字）把 woff2 压到 1–2MB，首屏更快；工具：`fonttools pyftsubset`。

## 4. 数据模型 & API 契约（阿里云）

内容即一份 JSON，结构等同 `miniapp/data.js`。小程序只读。

```ts
// GET https://你的备案域名/api/content  ->  200
interface Content {
  version: string;                 // 内容版本号，客户端比对决定是否刷新缓存
  ideas:   Idea[];                 // 6 条
  persons: Person[];               // 4 个
  books:   BookGroup[];
  wall:    string[];               // 6 句家规
}
interface Idea   { no:number; color:'coral'|'yellow'|'green'|'blue'; title:string; scene:string; key:string; body:string[] }
interface Person { id:string; tag:string; title:string; cta:string; intro:string; body:string[] }
interface BookGroup { who:string; items:{ t:string; n:string }[] }
```
> `key`/`body` 里允许 `<b>…</b>` 强调标记；小程序端用 `rich-text` 渲染或自己解析为高亮 `text`。原文见 `miniapp/data.js`，**首版可直接把它转成 `content.json` 当种子数据**。

### 客户端取数策略（重点：这就是"自动更新内容"的关键）
```ts
// 1) 先读本地缓存立即渲染（秒开）
const cached = wx.getStorageSync('content');
if (cached) render(cached);
// 2) 再请求 /api/content，version 变了就更新缓存并重渲染
wx.request({ url: API + '/api/content', success: r => {
  if (!cached || r.data.version !== cached.version) {
    wx.setStorageSync('content', r.data); render(r.data);
  }
}});
```
- **改文案 = 改阿里云上的 `content.json` → 客户端下次打开自动拿到新内容，不发版、不审核。**
- 离线/请求失败 → 用缓存或内置种子数据兜底，绝不白屏。

## 5. 阿里云这一端要做什么（最小后端）

这是只读内容，**不需要数据库、不需要复杂后端**。两种都行，推荐 A：

**方案 A（最简）：静态 JSON + Nginx**
```
/var/www/family-api/
  ├─ api/content        # 就是一份 JSON 文件（无扩展名或 content.json）
  └─ fonts/LXGWWenKai-Regular.woff2
```
- Nginx 配 HTTPS（必须）、给 `/api/content` 加 `Content-Type: application/json` 和短缓存头。
- 改内容 = SSH 上去改这个 JSON（或挂个 Git 钩子 / 简单 CMS）。

**方案 B（要可视化编辑）：Node/Nginx + 一个极简 admin**
- 加一个受密码保护的 `/admin` 编辑 JSON。非必需，按你后续需要再做。

### 三件必须在微信后台配置的事（个人主体能做）
1. **服务器域名白名单**：小程序后台 → 开发管理 → 服务器域名 → 把 `https://你的备案域名` 加进 **request 合法域名**。
2. **域名要求**：HTTPS + **已 ICP 备案**（你的阿里云服务器可备案，备案绑这个域名）。
3. `downloadFile` 合法域名：若用 OSS 域名存字体/图片，也要把 OSS 域名加进白名单。
> ⚠️ 个人主体能正常用 `wx.request` / `downloadFile`；**只有 `web-view` 用不了**——本方案全程不用它，无障碍。

## 8. 两端如何"自动更新"——给你（非 Codex）的结论

| | 网页端 (Vercel) | 小程序端 |
|---|---|---|
| **改内容（文案/书单/六条）** | push → 自动部署 | 改阿里云 `content.json` → 用户下次打开自动更新，**不审核** |
| **改功能（页面/交互/新模块）** | push → 自动部署 | 开发者工具上传 → **提交审核 → 发布**（数小时～1天），用户下次启动静默更新 |
| **后端** | 不需要 | 阿里云只放只读 JSON + 字体/图片，**不是传统后端** |

**一句话**：内容两端都能自动更新（小程序靠 API）；**功能代码**网页能自动、小程序必须走一次微信审核。让内容尽量数据化，就能把"提审"次数降到最低。
