微信小程序原型 · 个人主体

同一个家庭操作系统，
装进微信里。

个人主体不能用 web-view 套现有网页，所以整套界面按小程序原生组件重做：tabBar 导航、原生胶囊按钮、转发到聊天替代"发网址"、生成海报替代打印。右侧是可点的原型。

个人主体不能用 web-view
全部 原生页面，底部 tabBar：六条 / 给家人 / 共读 / 家庭墙。
小程序不能打印 A3/A4
家庭墙改为 canvas 生成海报 → 保存到相册。
不能外链 JS、整字库塞不进 2MB 主包
霞鹜文楷用 wx.loadFontFace 从阿里云加载；SVG 装饰转 PNG。
"把网址发给那个人"不成立
每个人物页用 原生转发到聊天，点开即停在对应入口。
代码改动要重新审核
文案走 阿里云接口（备案域名），改内容不发版、下次打开即更新。

原型为视觉与交互演示，真实工程用 Taro(React) 编译为小程序；详见随附的《小程序实现方案》。
