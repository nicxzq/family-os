# 爸妈 & 爷爷奶奶 内容规划

> 家里的"成年人受众"有两条线：**爸妈**（一起养娃的两个人）和**爷爷奶奶**（每天真正在带娃的人）。
> 这份文档把这两条线的目标、语气、页面和落地步骤写清楚。配套的代码改动见文末「落地步骤」。

---

## 结论先行

- **爸妈线**：`for-wife.html` 从"爸爸写给妈妈的信"改成"父母共读"（我们俩）。已落地。下一步不是加更多说教页，而是给父母一个**每月对齐的轻仪式**。
- **爷爷奶奶线**：从 `for-family-friends.html`（现在是"爷爷奶奶和朋友"混在一起）里**独立出来**，单独一页，针对隔代养育的真实场景——喂饭、零食、手机、比较、宠爱。**只讲故事 + 一句"我们能做什么"，不讲道理。**
- 两条线共用同一套**六条家规**底层，只是入口、语气、篇幅完全不同。

---

## 为什么要拆这两条线（第一性）

本质问题：**同一套理念，讲给"立场不同、注意力不同"的人，必须换完全不同的方式。** 主页那句话已经说了——"同一套想法，讲给不同年纪、不同立场的人，需要完全不同的方式。"

- **爸妈是"同学 / 合伙人"**：有决定权、能读长文、需要的是*对齐和共识*，以及让共识不散掉的*仪式*。给他们的东西要平等、坦诚，"我们"而不是"我要你"。
- **爷爷奶奶是"每天真正在带娃的人"**：决定权有限，但对日常影响巨大。他们不看长文、警惕说教，而且**出于爱**容易过度——追着喂饭、把零食和糖当爱、用手机哄娃、"你看别人家孩子"。
  - 给他们讲道理 = 制造对立；
  - 给他们讲故事 + 一个具体动作 + 给足面子 = 真正改变日常。

**对用户的影响**：拆开后，妈妈不再是被单独"说服"的对象（那本身就隐含不对等）；爷爷奶奶第一次有一页是真正对着他们的处境说话，而不是被夹在"和朋友一起"的通用故事里。

## 爸妈线（parents）

**目标：从"对齐一次"到"每月保持对齐"。**

- **P0 · 已完成**：`for-wife.html` 改成父母共读视角（"我们俩"）；`index.html` 的 hero 标签和卡片文案改成"给爸爸妈妈"。
- **P1 · 父母月度对齐**：复用 `weekly-review.html` 的思路，但做成*父母专属、私密、极简*的一页。三个问题就够：
  1. 这个月，我们自己在学什么？
  2. 孩子身上看到了什么变化？
  3. 下个月，少做一件什么"卷"的事？
- **P2 · 六条家规父母自检清单**：把六条家规做成给*父母自己*打勾的清单（不是给孩子的行为要求）。

**语气**：平等、坦诚、不说教。始终"我们"，避免"我要你/你应该"。

## 爷爷奶奶线（grandparents）

**目标：让最爱孩子的人，用不制造对立的方式，成为这套理念的一部分。**

- **P0 · 已建骨架**：`for-grandparents.html`。保留三个故事里最戳长辈的那条（"不读书的爸爸养出爱读书的孩子"），再补 2 个**隔代养育专属**场景。
- **内容原则**：
  - 不讲道理，只讲故事；每个故事配**一句"这件事我们能做的"**——正向、具体、给足面子。
  - 篇幅极短，读完不用回复。
  - **尊重优先**：绝不能让长辈觉得"被纠正"。把功劳给长辈——"孩子跟着您学会了……"。
- **隔代场景候选**（每个 = 一个小故事 + 一句能做的）：
  1. 追着喂饭 → 让他自己吃，饿一顿不要紧。
  2. 零食 / 糖当爱 → 换成"一起做一件事"当爱。
  3. 手机哄娃 → 十分钟陪玩，比一小时手机强。
  4. "你看别人家孩子" → 只跟昨天的他比。
  5. "读书没用" → 家里有人在看书，比说一百句都管用。
- **P1 · 重新定位 friends**：把 `for-family-friends.html` 收窄为"朋友版"，爷爷奶奶从主页导流到 `for-grandparents.html`。

## 约束 / 注意

- 内容必须忠于 `uploads/` 里的李笑来六原则：方向比目标重要、读书是家事、一起学不教训、软技能也是学问、保护彼此的注意力、不慌时间是朋友。
- 设计只用 `styles.css` 既有 token / 组件，不引入新色、新字体（`LXGW WenKai TC`）。
- 新页面非 gated（和 `for-family-friends.html` 一致）。若日后要登录门禁，运行 `node scripts/sync-gate.js`。
- `miniprogram/data/{stories,readers,topics}.js` 是生成文件，勿手改；`roles.js` / `role.wxml` 是手工维护。
